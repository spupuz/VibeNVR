## 2026-06-16 - [Path Traversal in User Avatars Cleanup]
**Vulnerability:** A directory traversal vulnerability existed in the user avatar cleanup logic (`backend/routers/users.py`) when users uploaded a new avatar. The application concatenated `/data` with the old avatar path without verifying the resulting path. If an attacker had tampered with the `old_avatar` DB record to include directory traversal sequences (like `../../`), the `os.remove` call could lead to arbitrary file deletion on the server.
**Learning:** File paths derived from user-controlled input or database records must be strictly validated before any file system operations like `os.remove`. Using standard `os.path.join` is insufficient if the concatenated suffix contains relative traversal sequences (`..`).
**Prevention:** Always resolve the final absolute path using `os.path.abspath(path)` and verify that it explicitly begins with the intended allowed root directory string (e.g., `abs_path.startswith("/data/avatars/")`) before performing any read or delete file operations.
